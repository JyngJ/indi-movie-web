## 지도 UX

### 검색 오버레이 고도화
- 구현 범위:
  - 현재 `MapView` 내부 검색 오버레이를 실제 검색 화면으로 확장한다.
  - 검색창은 `+/-` FAB와 같은 색상 계열을 유지한다: `--color-surface-card`, `--color-border`, `--color-text-body`.
  - 검색 오버레이 진입 시 모바일 키보드 자동 표시를 시도한다. 브라우저 정책상 사용자 제스처 없이 불가능할 수 있으므로, 현재처럼 검색 버튼 클릭 핸들러에서 dummy input focus 후 실제 input으로 넘기는 방식을 유지한다.
  - 뒤로가기 버튼은 현재 상태처럼 검색창 내부에 둔다.
- 최근 검색:
  - `localStorage` key: `movie:recent-searches:v1`.
  - 최대 8개 저장.
  - 동일 검색어는 중복 저장하지 않고 맨 위로 이동.
  - 항목 오른쪽 X 버튼으로 개별 삭제.
  - storage 접근은 private mode / 차단 설정을 고려해 `try/catch`로 감싼다.
- 자동완성:
  - 대상: 감독 이름, 영화 이름, 영화관, 지하철역, 동네 이름.
  - 후보 탭 시 입력창에 해당 텍스트를 채운다.
  - 초기 구현은 클라이언트 scoring으로 시작한다: 공백 제거, 소문자화, 포함 여부, prefix match, 간단한 유사도 점수.
  - 규모가 커지면 Supabase RPC + Postgres `pg_trgm` similarity / full text search로 이전한다.
- 결과 섹션 우선순위:
  - 가장 높은 점수의 후보 타입을 대표 타입으로 보고 해당 섹션을 최상단에 배치한다.
  - 역 검색: `지하철역` → `지역` → `근처 극장`.
  - 감독 검색: `감독` → `영화`.
  - 영화 검색: `영화` → `상영 중인 극장 최대 3개` → 3개 초과 시 `상영관 더보기`.
  - 지역 검색: `지역` → `지역/근처 극장`.
  - 극장 검색: `극장`.
- 터치 인터랙션:
  - 지역/역: 검색 오버레이를 닫고 해당 위치로 `flyTo(..., 15)` 이동.
  - 감독: 이번 단계에서는 `to be implemented`.
  - 상영 중 영화: 검색 오버레이를 닫고 지도에 `영화: [영화 이름] (X)` 필터 칩을 추가한다.
- 데이터 판단:
  - 현재 지도 데이터(`theaters`)만으로 가능한 것:
    - 영화관 검색.
    - 극장 주소에서 `마포구`, `종로구` 같은 구 단위 지역 후보 추출.
    - 지역 후보의 중심점은 해당 지역에 속한 극장 좌표들의 centroid로 근사.
  - 현재 지도 데이터만으로 정확히 불가능한 것:
    - 지하철역 검색, 역 좌표, 노선 태그, 역 기준 거리.
    - 동네(동 단위) 좌표와 행정구역 중심점.
  - 단기 타협안:
    - 새 DB 테이블/정적 역 데이터 없이 구현할 경우, 역 검색은 제외하거나 “극장 주소 기반 지역 검색”만 제공한다.
    - 동네 검색은 주소 문자열에서 추출 가능한 범위까지만 지원한다.
  - 정확한 역/동네 검색을 하려면 이후 `stations` / `areas` 데이터가 필요하다.
- 필요한 추가 query:
  - 전체 영화 목록 및 감독 목록.
  - 상영 중 영화별 극장 목록.
  - 위치 기반 거리 표시는 우선 클라이언트 haversine으로 처리하고, 데이터가 커지면 PostGIS RPC로 이전한다.
