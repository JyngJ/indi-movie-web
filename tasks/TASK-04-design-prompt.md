## 온보딩 디자인 요청 — 영화볼지도

첫 방문자에게 1회 노출되는 온보딩 4페이지를 디자인해줘. 기존 디자인 시스템(토큰·컬러·타이포)을 그대로 따르고, 새 스타일 언어를 만들지 마.

### 필요한 산출물

1. **모바일 온보딩 4페이지** (375px 기준) — 풀스크린 오버레이
2. **데스크톱 변형 1종** — 풀스크린이 아닌 중앙 모달 카드 형태 (4페이지 중 1페이지만 대표로)
3. **다크모드 버전** (4페이지 중 1페이지만 대표로)
4. 하단 도트 인디케이터 4개의 active/inactive 상태 포함

### 공통 레이아웃

- 우상단 "건너뛰기" 텍스트 버튼 (전 페이지, 터치 영역 44px 이상)
- 하단 도트 인디케이터 4개 (좌우 스와이프 전환 전제)
- 각 페이지 상단 60~65%는 비주얼 영역, 하단은 텍스트 영역
- 비주얼은 **일러스트**로 간다. 서비스의 핵심 모티프(지도 위에 떠 있는 영화 포스터 핀, 바텀시트, 극장)를 앱의 컬러 토큰 안에서 단순하게 그린 일러스트. 페이지 4개가 한 세트로 보이도록 일관된 스타일로. 사진·실사 스크린샷 아님.
- iOS safe-area(상단 노치, 하단 홈 인디케이터) 고려
- 서사: ① 다 모았다 → ② 뭘 볼지 → ③ 어디서 볼지 → ④ 위치 켜기 + 시작

### Page 1 — 후킹: 다 모았다

- 비주얼: 포스터 핀 여러 개가 떠 있는 지도 일러스트 (앱의 시그니처 장면)
- 헤드: **"전국 독립·예술영화관, 다 여기 모았어요"**
- 서브: "{N}개 극장의 상영 시간표를 매일 모아요. 이제 극장 인스타를 하나하나 뒤질 필요 없어요."
  - `{N}`은 실제 극장 수가 동적으로 들어가는 자리 (예: "155개"). 숫자 길이가 2~3자리로 변해도 줄바꿈이 안 깨지게.
- 최하단 작은 텍스트: "영화볼지도 — 독립·예술영화관 상영 시간표를 지도 한 장에"

### Page 2 — 상영작: 뭘 볼지

- 비주얼: 큐레이션 섹션 느낌의 일러스트 (가로로 늘어선 포스터 카드 행)
- 헤드: **"오늘 뭘 볼까, 고민부터 재미있게"**
- 불릿 3개 (아이콘+텍스트 리스트 형태 제안 환영):
  - 곧 내려가는 영화, 오랜만에 돌아온 영화를 먼저 알려드려요
  - 한 감독의 작품이 모이면 자동으로 특별전으로 묶여요
  - 칸·베니스·아카데미 수상작 중 지금 볼 수 있는 것만 골라서

### Page 3 — 지도: 어디서 볼지

- 비주얼: 포스터 핀 클로즈업 + 극장 바텀시트가 열린 장면 일러스트
- 헤드: **"영화를 고르면, 극장이 보여요"**
- 불릿 3개:
  - 핀의 포스터 = 지금 그 극장에서 상영 중인 영화
  - 영화 제목, 감독, 지하철역 — 뭘로 검색해도 찾아져요
  - 날짜를 바꿔가며 이번 주 상영을 미리 볼 수 있어요

### Page 4 — 위치 권한 + 시작 (CTA 페이지)

위치 켜기를 적극적으로 부탁하는 페이지. 부탁 → 보상 약속 구조를 시각적으로도 강조 (이 페이지만 헤드가 주인공).

- 비주얼: 내 위치 마커를 중심으로 주변에 극장 핀들이 모여드는 일러스트
- 헤드: **"어디에 계신지 알려주세요!"**
- 서브: **"가까운 극장부터, 지금 출발하면 볼 수 있는 상영부터 보여드릴게요."**
- 안심 문구 (서브 아래 작게): "위치는 극장을 찾는 데만 쓰고, 저장하지 않아요"
- **CTA 주 버튼** (프라이머리, 풀와이드): "위치 켜고 시작하기" → 탭하면 OS 위치 권한 팝업
- **CTA 부** (텍스트 링크 수준으로 약하게): "위치 없이 둘러보기" → 서울 기본 뷰
- 이 페이지만 도트 아래 CTA 영역이 추가됨 — 1~3페이지와 수직 리듬이 어긋나 보이지 않게 처리

### 제약·주의

- 톤: 기존 앱 서피스 톤 유지, 온보딩만 튀면 안 됨
- 텍스트는 위 카피 그대로 사용 (수정 제안은 별도 표기로 환영)
- 버튼·도트·건너뛰기 모두 기존 컴포넌트 토큰 재사용 가능한 형태로
- 페이지 스와이프 전환 애니메이션 컨셉 한 줄 제안 (구현은 CSS scroll-snap 예정)

## 카피 확정 근거 (구현자 참고, 프롬프트에 포함하지 않음)

- Page 1 헤드는 완전성 후킹("다 모았다" 패턴), 서브의 극장 수 `{N}`은 **하드코딩 금지** — theaters 데이터에서 동적 카운트로 주입 (구현 시 `useTheaters()` 또는 서버 조회).
- Page 4는 permission priming 패턴: 권한 요청 전에 가치(가까운 극장, 갈 수 있는 상영)를 먼저 약속. "저장하지 않아요"는 코드 검증 완료 — geolocation 좌표는 클라이언트의 거리 계산·정렬·지역 판별에만 쓰이고 analytics·Supabase·API 요청에 포함되지 않음 (2026-07-05 확인). 좌표를 서버로 보내는 기능이 추가되면 이 문구도 함께 수정할 것.
- 온보딩은 3페이지 → 4페이지로 확정 (위치 권한 priming 페이지 분리). `TASK-04-onboarding.md`의 페이지 스펙보다 이 문서의 카피가 우선.
